# AI翻訳時代のペイン・ポイント

AI翻訳ツールの急速な普及により、エンタープライズの言語業務担当者が直面する課題は変化しています。

問題は「翻訳ツールがない」ことではありません。\*\*「ツールが増えすぎて、どう組み合わせればよいかわからない」\*\*ことです。

***

## 経営層と現場の認識ギャップ

| 経営層の期待             | 現場の現実              |
| ------------------ | ------------------ |
| 「AIツールを入れれば自動化できる」 | 「ツール同士がつながらない」     |
| 「コストが下がるはず」        | 「どこにAIを使うべきかわからない」 |
| 「品質は維持できるはず」       | 「品質チェックが属人化したまま」   |
| 「ベンダーに任せれば大丈夫」     | 「ベンダーロックインが怖い」     |

***

## 6つのペイン・ポイント

1. [ワークフロー可視化の欠如](workflow-visibility.md) — 何がどこで起きているか見えない
2. [AI統合ポイントの不明瞭性](ai-integration.md) — どこにAIを入れるべきかわからない
3. [品質管理プロセスの属人化](quality-management.md) — 担当者の経験に依存している
4. [ワークフロー設計知識の不在](workflow-design-knowledge.md) — 設計方法論がない
5. [ベンダーロックインとシステム依存](vendor-lockin.md) — 身動きが取れない
6. [ナレッジ蓄積の停滞](knowledge-stagnation.md) — 知見が組織に残らない

***

これらの課題に対して、翻訳ラボがどう応えるかは[バリュープロポジション](../barypuropojishon/value-proposition.md)をご覧ください。
