# プラットフォーム機能概要

翻訳ラボのLTPは、6つのコア機能で構成されます。これらは独立したツールではなく、相互に連携するプラットフォームとして機能します。

---

## 6つのコア機能

| 機能 | 解決するペイン | 詳細 |
|---|---|---|
| [ワークフロー可視化ツール](workflow-visualization.md) | 全体が見えない | → |
| [AI統合設計フレームワーク](ai-integration-framework.md) | どこにAIを入れるか | → |
| [品質管理プロセス設計](quality-process.md) | 属人化している | → |
| [ワークフロー設計方法論](methodology.md) | 設計できない | → |
| [ベンダー中立型統合レイヤー](vendor-neutral.md) | ロックインされている | → |
| [ナレッジ蓄積・共有](knowledge-platform.md) | 知見が残らない | → |

---

## アーキテクチャ概要

```
フロントエンド
├─ ワークフロー可視化UI（インタラクティブ図）
├─ 設計テンプレートライブラリ
├─ ROI計算ダッシュボード
└─ コミュニティフォーラム

バックエンド（Dify / FastAPI）
├─ ワークフロー設計エンジン
├─ AI統合推奨ロジック
├─ 品質メトリクス計算
├─ ナレッジベース（RAG）
└─ API統合レイヤー

外部統合
├─ TMS各社API
├─ MT各社API
├─ 用語管理ツールAPI
└─ 品質チェックツールAPI
```

---

## MVP（Phase 1）の範囲

最初のリリースでは、最もニーズが高く、最も実証しやすい**ワークフロー可視化ツール**に絞ります。

- 現状ワークフローの図式化支援
- テンプレートベースの設計
- 簡易ROI計算機

この機能だけでも、「経営層に説明できない」という担当者の最大のペインに直接応えます。
